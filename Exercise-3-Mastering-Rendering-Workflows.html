Introduction

This exercise builds on the project setup, recording, and editing workflows covered in the previous exercises. Once you've created a well-edited compilation of your best takes, it's time to focus on the final stages of production in ReaClassical: mixing, mastering, and rendering your finished work.

Keyboard Shortcuts: This guide shows Mac shortcuts first with Windows equivalents in brackets. On the studio Mac, you'll need to press fn + the function key (e.g., fn+F8) unless the keyboard settings have been changed to use function keys without the fn modifier.
Understanding ReaClassical's Mixing and Mastering: In ReaClassical, mixing and mastering are integrated into a streamlined workflow. You work with mixer tracks, automation, and snapshots to balance your recording, then use the powerful CD marker creation tools to prepare your project for distribution. This integrated approach allows you to handle all aspects of finalising your project efficiently.

Part 1: Mixing with Automation and Snapshots

Mixing in ReaClassical: Classical music mixing typically focuses on natural tonal balance, proper spatial positioning, and transparent dynamics processing. ReaClassical provides two powerful approaches: traditional automation for gradual changes over time, and the Mixer Snapshot Manager for quickly recalling different mixer states at specific points in your project. Together, these tools allow you to create a polished, natural-sounding mix.
  1. Open your edited project:
    • Open the ReaClassical project from the previous exercise where you completed your editing
    • Ensure that your destination folder contains the complete, edited piece with all crossfades finalised
    • If you're using a vertical workflow, verify that all edits in the destination folder are complete before proceeding
  2. Understanding mixer tracks:
    • Note that all audio flows through the mixer tracks (labelled with "M:" prefix) to the RCMASTER bus
    • Open Mission Control (C) to view and manage your mixer tracks
    • Mixer tracks are where you adjust volume, panning, add effects, and control routing
    • Any changes made to mixer tracks affect all instances of those microphone channels throughout your project
    • By default, mixer tracks are only visible in the mixer panel, but you can make them visible in the arrange window via Mission Control if needed
  3. Using the Mixer Snapshot Manager:
    • Press Shift+M to open the Mixer Snapshot Manager
    • This tool allows you to create automatically-recalled mixer settings tied to specific items in your project
    • Click on an item in the destination folder that represents the start of a new section (e.g., the beginning of a different movement)
    • Adjust your mixer settings - faders, pan, effects parameters, mute/solo - to suit this section
    • Click "Set Snapshot..." in the Mixer Snapshot Manager window
      • Notice how the snapshot is now listed in the table, tied to that specific item
    • Move to another section that needs different mixer settings and repeat the process
    • During playback, with the window still open, the mixer will automatically recall the appropriate snapshot as each section plays
    Tip: For classical music, mixer snapshots are ideal for handling different movements or sections that have distinctly different balances - for example, a quiet solo passage followed by a full orchestral tutti. The automatic recall during playback makes it easy to verify that transitions sound natural.
  4. Converting snapshots to automation:
    • Mixer snapshots rely on real-time playback for auto-recall
    • Before rendering (especially faster-than-realtime rendering), you should convert snapshots to permanent automation
    • In the Mixer Snapshot Manager window, click "Convert to Automation"
      • Choose which folder to use for gap detection (this controls when the mixer transitions occur)
      • Decide whether to allow mid-gap switching
    • Notice how mixer tracks now appear in the arrange window with visible automation lanes
    • These automation points can be manually adjusted if needed using the pencil tool or by dragging points
    • If you later move items around, simply click "Convert to Automation" again to update the automation data
    • To return to snapshot mode for further editing, click "Clear Automation"
  5. Using volume automation on tracks:
    • For more detailed control beyond mixer snapshots, you can add automation directly to destination or source folder tracks
    • Select a track in the destination or source folder
    • Position your edit cursor where you want the automation to begin, or create a time selection for a specific region
    • Press I to open the automation dialogue
      • In edit cursor mode, the automation affects from that point forward to the end of the project
      • In time selection mode, the automation is limited to the selected region
    • Set your desired volume level and optional ramp in/out lengths
    • Click "Apply Automation"
    • Use this approach for bringing up spot microphones during solo passages, or reducing levels during louder tutti sections
  6. Working with advanced automation:
    • If you need to automate additional parameters beyond volume (such as FX parameters), enable "Advanced Mode" in the automation dialogue
    • This gives you access to all available envelopes for the selected track
    • To automate mixer tracks or special tracks, first make them visible in the arrange window:
      • Open Mission Control (C)
      • Check the box on the desired track row to make it visible
      • You can now select it and use I to add automation
  7. Adding effects and processing:
    • Open Mission Control (C) to add effects to mixer tracks
    • For classical music, consider adding:
      • Subtle EQ for tonal balance (RCChannelEQ or RCMasteringEQ)
      • Light compression if needed (RCCompressor with conservative settings)
      • Reverb on a dedicated aux track for added ambience
    • For mastering-grade processing on the final output, add effects to the RCMASTER track:
      • EQ for final tonal adjustments
      • Light limiting (RCLimiter) to control peaks while maintaining dynamics
    • Remember: less is more in classical music - aim for transparency rather than obvious processing

Part 2: Track Spacing and CD Marker Creation

CD Markers and Metadata: For classical albums with multiple movements or pieces, proper track spacing and metadata are crucial. ReaClassical's metadata editor automates the creation of CD markers, regions, and even room tone automation, making album preparation remarkably efficient. This single function handles everything needed for professional CD manufacturing, digital distribution, and archiving.
  1. Preparing items for CD track creation:
    • Identify which items in your destination folder should become CD track starts
    • For each track start, select the item and press Command+F2 [F2] to rename it
    • Enter the track name (e.g., "I. Allegro", "II. Adagio", "III. Scherzo")
      • If you want a visual CD player countdown into a track, start the name with an exclamation mark (e.g., "!III. Scherzo")
      • The countdown length is set in ReaClassical Preferences (Command+F5 [Ctrl+F5])
    • Leave all other items unnamed - ReaClassical will recognise these as continuations of the previous track
    • Alternatively, you can use the ReaClassical Notes window (N) to name items
    Tip: For an "Easter egg" hidden track in the CD pre-gap (before track 1), simply don't name the first item. This creates audio that's accessible on some CD players but doesn't appear in the track listing.
  2. Adjusting track spacing:
    • Before creating CD markers, ensure your tracks have appropriate spacing
    • To set uniform gaps between all tracks:
      • Press Command+Y [Ctrl+Y] and enter a desired gap length in seconds (typically 2-5 seconds)
      • All items will be automatically repositioned with your specified spacing
    • To reposition individual tracks:
      • Select the item that starts the track you want to move
      • Use Command+← [Ctrl+←] or Command+→ [Ctrl+→] to shift it left or right
      • Gaps and crossfades are preserved during this operation
    • For attacca movements (that should flow directly together), maintain the musical connection with crossfades rather than silence
  3. Clearing existing item names (if needed):
    • If you've been working in a horizontal workflow and have item names from previous work, you may want to clear them all first
    • Press Command+T [Ctrl+T] to remove all names from items on the destination parent track
    • Then name only the items that should be track starts as described above
  4. Adding room tone (optional but recommended):
    • For professional classical recordings, room tone creates natural transitions between tracks and avoids complete digital silence
    • Open Mission Control (C)
    • Click "Add Special Tracks" and select "Room Tone"
    • Drag your recorded room tone (ambient sound from the venue) into the room tone track
      • Alternatively, you can generate synthetic room tone using the template in the ReaClassical manual
    • Place the room tone across the entire album timeline
    • When you create CD markers (next step), ReaClassical will automatically generate volume automation for seamless crossfades between music and room tone
  5. Creating CD markers and metadata:
    • Press Y to open the Metadata Editor
    • The window displays all your named items as CD tracks
    • Enter album-wide metadata:
      • UPC/EAN code (if available from your label or distributor)
      • ISRC number pattern (if available) - ReaClassical can auto-increment for subsequent tracks
      • Album title
      • Performer name
      • Composer name
      • Genre (typically "Classical")
      • Year of production
    • You can also enter track-specific metadata by filling in the table rows
    • Click "Apply" or press Command+Return [Ctrl+Enter] to create the CD markers
    Tip: The metadata editor automatically creates: CD markers at each track start (with proper 2-second pre-gap before track 1), regions defining each track, room tone automation (if you added a room tone track), and a complete metadata file. If you later adjust track positions, simply press Y again to update everything instantly.
  6. Reviewing the created markers and automation:
    • After clicking "Apply", check the timeline ruler to see your CD markers
    • Notice the 2-second pre-gap before the first track (required by CD standard)
    • If you used exclamation marks, you'll see additional countdown markers
    • If you added room tone, check that volume automation has been created:
      • The room tone fades out as music begins and fades in during gaps
      • These crossfades are perfectly aligned to create seamless transitions
    • Regions have been created defining each track's extent
  7. Advanced metadata features:
    • For digital-only releases (no physical CD), check "Digital Release Only" in the metadata editor
      • This ignores CD frame alignment and offset requirements
      • Markers can be placed with sample-accurate precision
    • To manually adjust marker positions (for example, to avoid a cough before a downbeat):
      • Drag the CD marker to the desired position
      • Click "Add Marker Offsets" in the metadata editor
      • The offset is now stored with the item and will be maintained if you reposition items
    • To remove all manual offsets, click "Remove All Marker Offsets"

Part 3: Rendering and Exporting

Rendering Options: For your module assessment, you'll be creating a 48 kHz / 24-bit WAV file of your final edited recording (typically a single work of 7-12 minutes). However, this section provides an overview of the various rendering workflows available in ReaClassical, including tools designed for multi-track albums and CD manufacturing. Understanding these options will be valuable for future professional work.
  1. Setting appropriate loudness levels:
    Important Guidance on Loudness Targets:

    For classical music, integrated LUFS measurements typically range between -23 and -18 LUFS, though this varies considerably depending on repertoire and recording style. These values are indicative only - actual targets should be determined by analysing comparable professional recordings in the same genre.

    Chamber music might be quieter (around -23 LUFS), full orchestral works might be louder (around -18 LUFS), and solo piano or vocal works vary widely. Always reference well-regarded commercial recordings of similar repertoire to understand the loudness range and dynamic characteristics appropriate for your project.

    • Find well-regarded commercial recordings of similar repertoire and analyse their loudness using a meter:
      • Listen to professional recordings in the same genre
      • Note the loudness range and dynamic characteristics of your references
      • Use these as a benchmark for determining your own appropriate loudness targets
    • Use REAPER's built-in JS Loudness Meter to check your levels:
      • Right-click on the FX button of the Master track
      • Add "JS: Loudness Meter Peak/RMS/LUFS"
      • Play through your entire project from start to finish
      • Note the integrated LUFS measurement at the end
    • Once you're happy with the internal balance of your mix (the relative levels between different microphones), you can use the RCMASTER track fader as a gain staging device to achieve your target loudness
    • For taming aggressively loud sections, consider using slow-reacting compression with gentle settings rather than heavy limiting
    • Ensure peaks remain below -1dBTP (true peak) to avoid potential clipping during format conversion
    • If using a limiter on the Master track, best practice is to avoid producing any gain reduction with it - it should act only as a safety ceiling
    Tip: REAPER's render dialogue includes a powerful feature for automatically normalising to target loudness and true peak values. This can save significant time when you need to meet specific delivery requirements for streaming platforms.
  2. Using ReaClassical render presets:
    • Press R to open the render dialogue
    • Click on "Presets", then "All Settings" to see ReaClassical presets
    • Available presets include:
      • "ReaClassical - Album (16-bit WAV)" for CD-quality full album export
      • "ReaClassical - Album (24-bit WAV)" for high-resolution full album export
      • "ReaClassical - Album (FLAC)" for compressed lossless full album export
      • "ReaClassical - Individual Tracks (16-bit WAV)" for CD-quality separate track export
      • "ReaClassical - Individual Tracks (24-bit WAV)" for high-resolution separate track export
      • "ReaClassical - DDP" for professional CD manufacturing
    • Select the appropriate preset for your needs
    • The presets include proper dither settings for bit depth reduction
    • For your module assessment: Use "ReaClassical - Album (24-bit WAV)" preset (despite the name, this works for single pieces too) to create your 48 kHz / 24-bit WAV submission. Ensure your project sample rate is set to 48 kHz in project settings (P)
  3. DDP fileset creation (professional workflow - optional):
    • DDP (Disc Description Protocol) is the industry standard for professional CD manufacturing
    • Note: DDP creation is designed for multi-track albums. For your module assessment (typically a single work), you won't need this workflow, but understanding it is valuable for future album projects
    • To create a DDP fileset:
      • Select the "ReaClassical - DDP" preset in the render dialogue
      • Choose a destination folder
      • Click "Render" to create the complete DDP fileset
    • The fileset includes all necessary files for CD replication:
      • Audio data with proper CD framing
      • All track markers and CD-TEXT information
      • ISRC codes and other metadata
      • UPC/EAN barcode information
    • Most professional CD manufacturing plants prefer DDP over CD-R masters
    • DDP files can be verified before sending to ensure accuracy
  4. Dry-run rendering for verification:
    • Before committing to a full render, you can use REAPER's dry-run feature
    • In the render dialogue, click "Dry run"
    • This quickly analyses your project without creating files:
      • Provides integrated LUFS values
      • Shows true peak measurements
      • Reveals if any clipping will occur
    • Use this to verify loudness before rendering
    • Make any necessary adjustments to the RCMASTER fader (for overall gain staging) or limiting
    • Run another dry-run to confirm the results
    • This is much faster than rendering and re-rendering full files
  5. Individual track export (for reference):
    • This workflow is used when you have multiple separate tracks on an album that need to be exported as individual files
    • Note: Your module assessment typically involves a single work rendered as one continuous file - this section is included for completeness and future reference
    • Select the "ReaClassical - Individual Tracks (24-bit WAV)" preset
    • Verify that "Regions" is selected in the "Bounds" dropdown
      • This ensures each region created by the metadata editor becomes a separate file
    • Choose your destination folder
    • Click "Render" to create individual files for each track
    • Files will be automatically named based on your track titles from the metadata editor
    • These files are also useful for providing preview tracks to clients or for streaming services
  6. Dither for bit depth reduction:
    • When rendering to 16-bit (CD quality) from a 24-bit project, dither is essential
    • Dither prevents quantisation distortion when reducing bit depth
    • ReaClassical presets include appropriate REAPER dither settings by default
    • Alternatively, you can use RCDither:
      • Add RCDither as the final plugin on the master bus
      • Select "16-bit" from the dropdown
      • Disable REAPER's built-in dither in the render dialogue to avoid double-dithering
    • Note: RCDither uses the "Not Just Another Dither" algorithm, which is considered one of the finest available
  7. Creating CUE files and album reports:
    • After rendering your album, check your project folder for automatically-generated files:
      • A CUE file for use with your rendered audio file
      • An HTML album report with formatted track listings and metadata
      • A plaintext album report with the same information
      • A complete technical metadata file (useful for sending to manufacturers)
    • These files are created when you press Y to open the metadata editor
    • They update automatically each time you make changes and apply them
    • The album reports are valuable for:
      • Providing timing information to clients and labels
      • Including in liner notes or documentation
      • Archiving session details

Part 4: Project Management and Archiving

Project Archiving: This section covers suggested good practices for archiving your classical recording projects to ensure you can return to them in the future if revisions are needed. While not mandatory, following these practices will help with organising files, creating backups, and documenting your work. ReaClassical's project structure and integrated notes system make this process straightforward.
  1. Document your project:
    • Open the project notes by pressing P (Project Settings)
    • Navigate to the "Notes" tab
    • Add detailed information about the recording:
      • Recording date and location
      • Performers, conductor, and instruments
      • Microphone placement and equipment used
      • Any special techniques or challenges encountered
      • Processing and effects applied
      • Target loudness and delivery formats
    • Use ReaClassical Notes (N) to add notes to specific items, tracks, or markers
      • This is helpful for documenting editing decisions or problem areas
    • This documentation will be invaluable if you need to revisit the project months or years later
  2. Clean up the project folder:
    • To reduce project size, you can remove unused media files
    • Go to File › Clean current project directory
    • Review the list of unused media files carefully
    • If you're certain you don't need these files, allow REAPER to move them to the Trash/Recycle Bin
    • Important: When in doubt, keep everything! Storage is relatively inexpensive compared to the value of your recordings
    • Consider keeping all source takes even if unused - you may want them for alternate versions later
  3. Create a complete project backup:
    • To create a fully self-contained archive of your project:
    • Go to File › Save project as...
    • Choose a descriptive name that includes the date (e.g., "Brahms_Quartet_Op51_Final_2026-02-25")
    • Check "Create subdirectory for project"
    • Check "Copy all media into project directory"
    • This creates a complete, portable project folder with all necessary media files
    • The resulting folder can be archived on external storage without dependencies on other locations
  4. Organise your renders:
    • Create a well-organised folder structure for your rendered files:
      • "Master" folder for the highest quality full mixes (24-bit WAV)
      • "CD" folder for DDP filesets and 16-bit WAV files
      • "Distribution" folder for various formats needed for streaming platforms
      • "Individual_Tracks" folder for separated tracks
    • Include the album reports (HTML and plaintext) with your deliverables
    • Consider including a PDF of the score if available, especially for premiere recordings
    • Add any relevant performance notes or programme notes
  5. Archive to external storage:
    • Copy your complete project folder and all renders to at least two separate external storage devices
    • Use different types of storage for redundancy:
      • One copy on an external SSD or hard drive
      • One copy on cloud storage (Dropbox, Google Drive, etc.)
    • Store one copy offsite if possible to protect against physical damage, theft, or disasters
    • For extremely important projects, consider creating a checksum file to verify data integrity over time
    • Document your backup locations and dates in your project notes
    • Test your backups periodically by opening archived projects to ensure they're still accessible

Summary

This exercise has introduced ReaClassical's integrated mixing, mastering, and rendering workflows. You have learned to:

  • Use the Mixer Snapshot Manager to create auto-recalled mixer scenes for different sections
  • Apply volume automation to destination and source tracks for detailed control
  • Convert mixer snapshots to permanent automation for faster-than-realtime rendering
  • Create properly spaced CD tracks with appropriate naming
  • Use the Metadata Editor to generate CD markers, regions, and room tone automation automatically
  • Render in various formats using ReaClassical presets, including professional DDP filesets
  • Document and archive projects for long-term preservation

Key Points

  • Mixer snapshots (Shift+M) are ideal for section-by-section mixing
  • Press Y to create CD markers, regions, and metadata in one step
  • Analyse comparable professional recordings to determine appropriate loudness targets for your repertoire
  • Use dry-run rendering to verify loudness before creating final files
